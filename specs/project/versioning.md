# Versioning Rules

## Version formats

### Tagged releases (stable)

Format: `v<major>.<minor>.<patch>`

Examples: `v0.1.0`, `v1.0.0`, `v1.2.3`

Rules:

- Use semantic versioning
- Tag in git with `v` prefix
- Major: breaking changes
- Minor: new features, backward compatible
- Patch: bug fixes
